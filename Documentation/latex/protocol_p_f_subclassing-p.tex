\subsection{Detailed Description}
If a subclass of $<$\+P\+F\+Object$>$ conforms to {\ttfamily \hyperlink{protocol_p_f_subclassing-p}{P\+F\+Subclassing}} and calls $<$register\+Subclass$>$, \hyperlink{interface_parse}{Parse} framework will be able to use that class as the native class for a \hyperlink{interface_parse}{Parse} cloud object.

Classes conforming to this protocol should subclass $<$\+P\+F\+Object$>$ and include {\ttfamily P\+F\+Object+\+Subclass.h} in their implementation file. This ensures the methods in the Subclass category of $<$\+P\+F\+Object$>$ are exposed in its subclasses only.
